# Qué es un Shell

**Tags**:: #lenguajes/shells 
**Links**:: 

Un *shell* es un tipo de programa de computadora conocido como command-line interpreter o intérprete de líneas de comando. Es parte de un sistema operativo y permite a los usuarios controlar dicho OS a través de interfaces de líneas de comandos. El primer shell fue desarrollado por [Louis Pouzin](https://en.wikipedia.org/wiki/Louis_Pouzin) en 1960, que lo llamó shell porque es la capa más exterior y delgada de un sistema operativo.

La práctica de *shell scripting* (o programación de consola) no es un sólo idioma. Hay diferentes dialectos de shell scripting y cada uno de ellos se considera un leguaje. Debido a que muchos de ellos usan comandos de lenguaje natural, se consideran fáciles de aprender, incluso sin un experiencia en la programación. Sin embargo, para tareas más complejas, los shells requieren mucha práctica.

El shell scripting es una de las maneras más simples de configurar automatizaciones, le da a científicos de datos, DevOps, y TechOps estructuras condicionales y de bucles para repetir comandos con solo ejecutar el archivo donde dichos comandos se encuentran escritos.

Los shells tienen dos categorías:

- **Shells de Línea de Comandos:** Se accede a ellos a través de interfaces de líneas de comandos donde el sistema recibe el input del usuario a través de comandos legibles para humanos y entonces usa la interfaz de línea de comandos para mostrar el output.
- **Shells Gráficos:** Estos shells usan una interfaz gráfica de usuario, graphical user interface o GUI, para llevar acabo acciones y operaciones esenciales como abrir, cerrar y guardar archivos.

### Alternativas a los Scripts de Shell
Generalmente, se entiende que el término *shell* se refiere a aquellos programas diseñados para sistemas tipo Unix y que los lenguajes de scripting de otros sistemas operativos son alternativas a shell, como *EXEC* para el OS VM de IBM o los archivos *batch* (terminación `.bat`) de Windows.

### Desventajas de Shell
En general, los shells no están diseñados para tareas grandes o demasiado complejas. Por lo tanto, es fácil cometer errores y son difíciles de diagnosticar y arreglar (debug); además, tampoco ofrecen mucho en cuanto a estructuras de datos.
